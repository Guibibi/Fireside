# Fireside Rebrand Plan

## Context

The project's GitHub repo is already named "Fireside" and the README uses the Fireside name, but all internal code, configs, and deployment scripts still use "yankcord" / "Yankcord" / "YANKCORD". This plan renames every occurrence to "fireside" / "Fireside" / "FIRESIDE" for a complete, clean rebrand. The live `server/.env` file is intentionally excluded.

**Scope:** ~38 files + 1 directory rename, ~133 individual edits.

---

## Phase 1: Rust Server (build-critical)

| File | Change |
|------|--------|
| `server/Cargo.toml` | `name = "yankcord-server"` -> `"fireside-server"` |
| `server/src/main.rs` | Default RUST_LOG `yankcord_server` -> `fireside_server`, startup message `"Yankcord"` -> `"Fireside"` |
| `server/src/routes/embed_routes.rs` | User-agent `YankcordBot/1.0 (+https://yankcord.local)` -> `FiresideBot/1.0 (+https://fireside.local)` (2 occurrences) |
